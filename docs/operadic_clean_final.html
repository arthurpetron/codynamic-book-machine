Operadic Composition Laws
1. Identity
idA ∘ f = f = f ∘ idB
For any operation f : A → B, composing with identity preserves the operation. Identity agents pass data unchanged.
2. Associativity
(h ∘ g) ∘ f = h ∘ (g ∘ f)
For composable operations f : A → B, g : B → C, h : C → D, grouping doesn't affect the result. Agent pipelines can be factored freely.
3. Type Safety
f : A → B, g : B → C ⟹ g ∘ f : A → C
Operations compose only when output type of f matches input type of g. The type system enforces correct composition at compile time.
4. Operadic Substitution
Given f : A₁ × A₂ × ... × Aₙ → B and gi : C → Ai
We get f(g₁, g₂, ..., gₙ) : C × C × ... × C → B
Multi-input operations can have their inputs substituted with other operations. This is the key property that distinguishes operads from categories.
